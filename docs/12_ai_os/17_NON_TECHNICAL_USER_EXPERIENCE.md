# Non-Technical User Experience Requirements

---

## Overview

Forge must be usable by:

→ users with ZERO technical background

The system must abstract all complexity.

---

## Core Principle

User interacts ONLY with:

→ conversation

User must NOT interact with:

- code
- files
- system internals
- APIs

---

## Communication Style

Forge must:

- use simple language
- avoid technical jargon
- explain concepts clearly
- adapt to user's language level

---

## Language Support

Forge must support:

- Arabic (primary)
- English
- mixed language (Arabic + English)

---

## Interaction Model

User flow must feel like:

→ talking to a smart human

NOT:

→ using a tool

---

## Guidance Behavior

Forge must:

- guide the user step by step
- ask relevant questions
- suggest improvements
- highlight risks

---

## Decision Support

Forge must:

- simplify choices
- avoid overwhelming the user
- present clear options
- recommend one option

---

## Error Handling

Forge must:

- explain errors clearly
- avoid technical details
- propose simple solutions

---

## Feedback Loop

Forge must:

- continuously check user satisfaction
- adapt responses
- refine understanding

---

## Progress Visibility

User must understand:

- what is happening
- what is completed
- what is next

WITHOUT seeing technical internals.

---

## Trust Building

Forge must:

- be consistent
- be transparent
- be reliable
- admit uncertainty when needed

---

## Strict Rule

User must NEVER feel:

- confused
- overwhelmed
- lost

Forge is responsible for clarity.
